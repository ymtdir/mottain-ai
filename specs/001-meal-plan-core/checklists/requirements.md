## 備考

- スコープ: US1（在庫→献立・買い物リスト）／US2（アレルギー・苦手の絶対回避）／US3（対話での献立変更）／US4（会話からの好み学習）。分量調整・提案根拠の可視化・5段階評価インターフェース・レシピ保存・イラスト生成は後続 spec に分離。
- 好み学習の入口は会話フィードバックに限定。5段階評価は同じ「好み学習の核」に後続 spec で足す前提。
- 回避制約（DietaryConstraint, ハード）と好みプロファイル（PreferenceProfile, ソフト）は概念上の兄弟＝「永続化されたユーザー文脈が提案を制約・調整する」もの。両者が競合する場合は回避を優先（FR-023）。実装アプローチ（メモリ保持・コンテキスト注入など）は plan/ADR で決める。
- NEEDS CLARIFICATION（要確認）なし。ユーザー入力が十分に具体的なため、既定値は前提（Assumptions）に明記した。
- 実装は plan/tasks で段階を切る想定: ①チャット疎通（LLM 往復の土台）→ ②在庫の解釈・構造化 → ③献立・買い物リスト生成 → ④回避ガードレール → ⑤対話変更 → ⑥好み学習。画面構成は plan で設計する。
